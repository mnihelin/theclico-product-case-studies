# n8n Data Extraction & Excel Export PoC

## Project Type

Automation / n8n / Data Extraction / Excel Export / PoC

## Overview

This case study summarizes my contribution to an n8n-based automation PoC developed at TheClico.

The project was designed to test whether simple automation workflows could accelerate repetitive data extraction processes. The main idea was to allow users to enter a supported application link and receive the extracted data as an Excel file.

The workflow focused on automating the process of collecting data and saving the output in a structured Excel format.

## Problem

Data extraction tasks can become repetitive and time-consuming when they are performed manually.

For product development and testing processes, teams may need to collect data quickly from supported sources and organize it in a usable format. Manually collecting this data and preparing Excel files can slow down experimentation and early-stage product testing.

The challenge was to test whether n8n could simplify this process through a basic automation flow.

## Solution

The PoC was designed around a simple user-facing flow.

Users could enter a supported application link. After the link was submitted, n8n handled the data extraction process in the background and generated an Excel file as the output.

The purpose was to test whether this type of automation could reduce manual effort and support faster data preparation during product development and PoC processes.
